# Color Palette & Roles

## Token Architecture (Figma 기준)

Figma 파일의 컬러 토큰은 3계층으로 구성된다:

```
Color (Primitive)  ← 원천 팔레트. 사용되는 모든 컬러
  ├─ Gray          (11단계: 50–950)
  ├─ Red           (11단계)
  ├─ Orange        (11단계)
  ├─ Yellow        (11단계)
  ├─ Green         (11단계)
  ├─ Blue          (11단계)
  ├─ Blue2 (JK)    (11단계)
  └─ Violet        (11단계)
        │
        ▼
Theme
  └─ Brand         ← JK/AM 모드에 따라 brand50–brand950 전환
        │
        ▼
Semantic           ← 용도별 최종 토큰. 코드에서 직접 참조
  ├─ Background    ← 화면 배경, 모달/팝업, 바텀시트 등 bg
  ├─ Typography    ← 텍스트 컬러
  ├─ Icon          ← 아이콘 컬러
  ├─ Fill          ← 버튼, chip, label, hover 등 컴포넌트 내부
  ├─ Border        ← 경계선, 구분선
  └─ Shadow        ← 그림자 (elevation용)
```

## Brand (Theme-Dependent)

브랜드 색상은 테마 토큰을 통해 JK/AM 간 전환된다. 동일한 토큰 이름이 완전히 다른 색을 가리키므로, 코드에서는 항상 토큰 이름을 사용해야 한다.

| Token | JK (Blue) | AM (Orange) | Role |
|-------|-----------|-------------|------|
| brand50 | `#EEF3FF` | `#FFF6EE` | 가장 연한 브랜드 틴트 — 미묘한 배경 |
| brand300 | `#82AAFF` | `#FFAB5E` | 중간 밝기 액센트 — 보더, 하이라이트 |
| brand500 | `#1B55F6` | `#FF6D12` | **Primary** — CTA, 링크, 핵심 UI |
| brand700 | `#0D2EA0` | `#9A3600` | 다크 액센트 — 눌림 상태, 강조 |
| brand950 | `#030926` | `#230800` | 가장 진한 브랜드 — 다크 모드 서브틀 필 |

## System Colors

상태 전달 전용. 장식이나 브랜딩 목적으로 사용 금지.

**Blue — 정보(Info)**
`#EBF5FF` · `#CCE4FF` · `#A4CEFF` · `#2684FF` · `#002F6E` · `#001C46`

**Red — 위험/에러(Danger/Error)**
`#FFF2F1` · `#FFD9D6` · `#FFB4B0` · `#F22A23` · `#630808` · `#3D0403` · `#210102`

**Green — 성공(Success)**
`#EEFFF6` · `#C4F5DF` · `#88E8BA` · `#02B160` · `#003D1E` · `#00240E`

**Yellow — 경고(Warning)**
`#FFFCE8` · `#FFF5CC` · `#FFE87A` · `#FFBB00` · `#6C3E00` · `#332500`

**Violet — 보조 액센트**
`#F5F3FF` · `#EDE8FF` · `#D8CDFF` · `#7C3AED` · `#2E0C6A` · `#1D004F`

## Base Neutrals
- **Black** `#131618` — 순흑이 아닌 웜-쿨 차콜. 눈이 편안한 다크 베이스
- **White** `#FFFFFF` — 순백
